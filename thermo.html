Thermo Fisher Scientific – Data Governance & Alation Catalog

LP Data Governance Analyst Intern II (Onsite) · May 2024 – Aug 2024

Data Governance Data Dictionary Metadata Management Stakeholder Enablement Excel Power BI Alation

Summary

Spearheaded the improvement of an early-stage LPG data dictionary and evolved it into a stakeholder-ready data discovery resource. The project progressed from a refined, Excel-based glossary (supported by Power BI for metadata collection) to a published catalog experience in Alation, enabling Supply Chain stakeholders to more easily discover, interpret, and use product metadata.

The Problem

Stakeholders lacked a consistent, self-service way to understand product data—what fields meant, where they originated, and how they should be interpreted. An early-onset data dictionary existed, but it required significant restructuring and refinement to support reliable data discovery and day-to-day use.

What I Did

  • Developed and implemented a fully functional LPG Data Catalog glossary using Excel as the primary data dictionary, mapping product attributes to source systems to improve data discovery for Supply Chain stakeholders.
  • Used Power BI during the development process to collect, validate, and organize necessary metadata, helping identify attribute usage patterns and gaps in the existing dictionary.
  • Evaluated an early-stage version of the data dictionary to identify structural, formatting, and usability weaknesses impacting metadata interpretation.
  • Restructured the glossary with reorganized attribute groupings, color coding, and enhanced accessibility, significantly improving stakeholder engagement and understanding of LPG metadata.
  • Iterated over multiple feedback cycles with the data governance team to resolve ambiguity and ensure the dictionary was stakeholder-ready.
  • Partnered with colleagues to publish the finalized glossary in Alation, enabling an always-available, self-service data discovery experience.

Process / Pipeline

  1. Assess existing dictionary → identify gaps and usability issues
  2. Restructure & standardize → improve organization, naming, and readability
  3. Iterate with stakeholders → refine definitions and resolve open questions
  4. Finalize Excel dictionary → clean, consistent staging artifact
  5. Publish in Alation → searchable, centralized data discovery platform

What Is Alation?

Alation is an enterprise data catalog and governance platform that enables organizations to scale data initiatives by making metadata searchable, standardized, and easier to understand. It provides a centralized location for discovering data assets, reviewing definitions, and understanding source-system context.

Publishing the LPG data dictionary in Alation moved the resource beyond a static spreadsheet to an encyclopedia-style, always-available experience for stakeholders.

Impact

  • Improved data discovery by transforming an early-stage dictionary into a structured, navigable resource.
  • Increased accessibility and adoption by publishing through Alation instead of relying on static files.
  • Supported stakeholder alignment by reducing ambiguity and improving metadata clarity.

Skills Demonstrated

  • Data dictionary design and standardization
  • Stakeholder collaboration and requirement gathering
  • Excel-based data modeling and documentation
  • Power BI for metadata exploration and validation
  • Enterprise data catalog publishing (Alation)
