UTK Haslam College of Business – Graduate Course Development: Missing Data & Imputation

Graduate Course Development · Business Analytics & Statistics · University of Tennessee

R mice (Multiple Imputation) Data Cleaning Synthetic Data Generation Model Evaluation (RMSE) Curriculum Development

Summary

Collaborated with faculty to develop a graduate-level course focused on diagnosing and handling missing data in applied business analytics. I contributed to course design, built a realistic synthetic dataset with thousands of records and inter-variable dependencies, supported missingness simulation and imputation workflows (including MICE), and helped establish a model-based evaluation framework so students could compare approaches using objective metrics.

Highlight: I contributed to building and piloting this graduate course while I was a full-time undergraduate student at the University of Tennessee, Knoxville.

The Problem

In real analytics work, missing data is unavoidable. If missingness is not diagnosed and handled correctly, it can introduce bias, reduce model performance, and lead to incorrect business decisions. The goal of the course was to teach students a practical workflow: diagnose missingness, clean data, apply appropriate imputation methods, and evaluate downstream model performance.

What I Did

  • Reviewed and refined a course data dictionary by identifying variables, data types, and logical dependencies between features (how A affects B, B affects C, etc.), helping inform the course’s structure and learning flow.
  • Built a synthetic dataset (10–15k records) representing season ticket holder behavior, including realistic dependencies across variables (e.g., ticket type linked to income, household structure, tenure, and account type).
  • Supported course design in R by developing and refining a structured course outline and instructional materials (e.g., an RMarkdown “master” file with lessons, code examples, and exercises).
  • Helped implement missingness experiments (simulating missing data) and guided students through imputation methods ranging from simple single imputation (mean/mode) to multiple imputation using mice.
  • Integrated a model-based evaluation approach where students compared imputation strategies by training predictive models and measuring accuracy against known ground-truth values (e.g., using RMSE).
  • Helped define a consistent scoring framework to compare student solutions and identify best-performing approaches relative to a reference (“parent”) benchmark.

Course Workflow (What Students Practiced)

  1. Understand the data → data dictionary, variable types, dependencies
  2. Clean the data → fix typos, inconsistent categories, and data quality issues
  3. Diagnose missingness → patterns, mechanisms, and practical implications
  4. Impute → single imputation (mean/mode) and multiple imputation (MICE)
  5. Model & evaluate → train models and compare results with objective metrics (e.g., RMSE)
  6. Select a final approach → choose the best-performing workflow and justify it clearly

Example Modeling Context

A core course use case was predicting whether a season ticket holder would renew (churn vs. retain) using the available features. Students learned how imputation choices affected model performance and how to defend their chosen approach with evidence.
